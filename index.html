Preface

　僕は長い時間Rubyをさわってきたので、自分の考えを早くコードに落とせるのはRubyです。 しかし、全ての問題に対してRubyが万能というわけではありません。 Rubyはそもそも大規模なプロジェクトには向いていない面があります。 新しい言語はいつでも歓迎し、大好きな言語であっても古くなればお別れする。そんな覚悟で日々プログラミングに励んでいます。 僕にこの考え方を与えてくれた John Z. Sonmez 氏と、常に進化し続ける言語たちに深く感謝の意を表します。

Skills

プログラミング言語は多くの言語から影響されて作られた経緯があるため、幾つかの言語を同時に勉強するのが効率がいいかなと思っています。 もちろん勉強した全ての言語を使いこなせるという訳ではないですが、勉強した中で特に好きな言語（~= 得意な言語）について紹介します。

Languages

Ruby

純粋なオブジェクト指向・省略できる括弧・メタプログラミングの柔軟性 の三拍子が揃っているため、 プロンプト上においては最も効率的にプログラムが書ける言語の1つだと思っている。 優秀な正規表現を持つ点もポイント。

Elixir

Ruby風な構文で書ける関数型言語。ErlangのVM上で動く。 関数型言語の考え方と、Erlangのシステムと、Rubyのように扱えるオブジェクトが合わさった素晴らしい言語。 macroによるメタプログラミングも強力。

Python

「やりかたは明確な一つの方法にすべき」という考え方が強い。 特に、関数やクラスに対して docstring を書ける点が気に入っている。 機械学習プログラミングのときによくお世話になっている。

CoffeeScript

JavaScriptをRubyやPython風な構文で書ける言語。 全ては式である（Everything is an Expression）からわかる通り、 ほとんどのCoffeeのコードがJavaScriptのコードに対応して変換されるため、 JavaScriptを書くときのテクニックの勉強に大いに役立った。

Shell

様々な環境でデフォルトで用意されており、POSIXによって仕様変更がほとんどないので、長期の運用に向いている言語。 awk, sed, make (Makefile) はGNU拡張を除いて使える（FreeBSD派）。 目標は「コピー一発、デプロイ完了」

Environment

Atom

Github製のエディタ。スニペットやパッケージの追加が楽なのでメインのiMacで使用している。

Git

バージョン管理システム。Ubuntuではコマンドから、MacOSではSourceTreeを通して使用している。

GitHub

Gitのバージョン管理を複数人で共有するためのWebサービス。 クラス企画の作業はGithubを通して行っている。

Sublime

どこにでも持ち運び可能な軽量エディタ。乏しい環境下でもコーディングのパフォーマンスを落とさずに済む。

Heroku

RubyやNode.jsなどが動くサーバを無料で提供しているサービス。 不便な部分もあるが、とりあえずこれで我慢している。

「Ecoris」は高専3年生の学園祭（工嶺祭）のクラス企画で展示した LeapMotion で行う3Dテトリス。 Ecoris は 3Dテトリス + エコ に由来し、手の動きを利用して缶ブロックを潰したり、 生ごみブロックを振って小さくしながら遊ぶテトリス。 このプログラム（C#）部分を担当し、工嶺祭の「学生会長賞」を勝ち取った。 使用環境は Unity + C#

「Recursive Tree Renderer」は、ランダム性を有する再帰木を描画するツール。 作成した再帰木は、背景を無色透明で保存することができる。 再描画ごとにランダムな樹形が描かれるので、美形が出るまで何回も描画させたり、各パラメータを変更しつつ遊んでもらえると嬉しいです。 使用環境は p5.js
